# Git Version Control

1. Install software
   1. Sign up for GitHub
   2. Install Git
      - Use "main" as the default branch
   3. Install GitKraken
   4. In GitKraken, configure name and email.
   5. Configure VS Code

2. First commit
   1. Review version control, versions, features
   2. Explain "commit":
      - Verb
        * Promise to do something "Politician commits to make Taiwan’s air cleaner"
        * To perform (or do) a crime "The bad guy committed murder"
      - Noun (programmer)
        * A snapshot showing the changes since the snapshot before
   3. Add all files in VS Code.
   4. Make "First commit" commit in VS Code.

3. Second commit
   1. Add VS Code workspace.
   2. Make "Add workspace" commit.

4. Third commit
   1. Add footer element
   2. Add paragraph "Date: March 11, 2026"
   3. Add paragraph "Made by: name".
   4. Add paragraph "This webpage uses HTML, CSS, and JavaScript."
   5. Make "Add footer" commit.

5. Review git log changes
   1. Open VSCode’s Git tab
   2. Click on school.html to look at diff

6. Introduce remote repository concept
   1. Review "commit" concept.
   2. "Repository" means storehouse, container. Repository is a container for snapshots (commits).
   3. Introduce "remote" repository, push, pull. Local repository (container) on your computer vs. Github repository (on someone else’s computer)

7. Push to GitHub
   1. Click `…` button next to SOURCE CONTROL and show "Repositories".
   2. Click the "Publish to GitHub" button
   3. Type "week1" in the field and click "Publish to Github public repository" text.
   4. Click "Open on GitHub" button in alert message.

8. Publish website
   1. Go to github.com and find `week1` repository
   2. Go to "Settings" in top nav
   3. Go to "Pages" in sidebar nav
   4. Change "Branch" to "main"
   5. Click "Save"
   6. Wait for "Your site is live at" message.
   7. Click link add "school.html" to URL.

9. Add default page for a folder
   1. Rename "school.html" to "index.html"
   2. Make commit
   3. Push to GitHub
